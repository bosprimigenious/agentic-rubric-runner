# 0.5.2.4 Web Evidence Policy

## Scope / 范围

Define how internet access may be added without turning `agentic-rubric-runner`
into a LoopPilot-style crawler or autonomous research agent.

定义如何加入联网能力，同时避免把 `agentic-rubric-runner` 变成 LoopPilot 式爬取器或自治研究 agent。

## Product Boundary / 产品边界

LoopPilot may proactively search, monitor, and collect materials for long-running
project work.

LoopPilot 可以为长期项目主动搜索、监控和收集资料。

`agentic-rubric-runner` should only fetch web evidence when the rubric explicitly
declares a source policy. Web evidence is part of scoring provenance, not a new
task-discovery loop.

`agentic-rubric-runner` 只有在 rubric 明确声明来源策略时才应抓取联网证据。联网证据是评分溯源的一部分，不是新的任务发现循环。

## Source Guidance / 来源建议

Preferred sources should be official, primary, or canonical:

首选来源应是官方、第一手或权威索引：

- company filings: `sec.gov`, official investor-relations pages;
- government and policy: official `.gov` sites and regulator pages;
- standards: `iso.org`, `w3.org`, `ietf.org`, `nist.gov`;
- software and APIs: official docs, release notes, GitHub repositories, package registries;
- academic: DOI landing pages, arXiv, PubMed, publisher pages;
- security: NVD, CISA, vendor advisories, CVE records.

## Acceptance / 验收

Offline acceptance must pass without network access:

```bash
scripts/acceptance.sh offline
```

Live web-enabled acceptance is a later gate and must not be required for normal PRs.

联网 live acceptance 是后续门控，不应成为普通 PR 的必要条件。

## Status / 状态

This iteration is a design contract. It does not yet implement browsing.

本迭代是设计合同，尚未实现联网浏览。
