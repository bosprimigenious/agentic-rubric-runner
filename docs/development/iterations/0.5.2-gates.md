# 0.5.2.x Gate Architecture

## Scope / 范围

Build the project-level acceptance structure before calling the project READY:

在宣称项目 READY 前，先搭好项目级验收结构：

- one local and CI gate entrypoint;
- real benchmark manifest;
- baseline regression contract;
- configurable domain gate;
- documentation and iteration-log structure.

## Version Position / 版本位置

This slice belongs to the fine-grained `0.5.2.x` line:

本切片属于细粒度 `0.5.2.x` 线：

| Internal version | Meaning |
|---|---|
| `0.5.2.1.x` | offline acceptance entrypoint and fail-closed manifest gate |
| `0.5.2.2.x` | domain gate configuration |
| `0.5.2.3.x` | baseline regression gate |

The public release target can still be `0.5.2` if these slices are bundled
together.

如果这些切片一起发布，公开发布版本仍可使用 `0.5.2`。

## Acceptance Entry / 验收入口

Canonical command:

```bash
scripts/acceptance.sh
```

When `DEEPSEEK_API_KEY` is unavailable, the script runs deterministic checks and
the offline gate for local or PR use. Release mode requires `DEEPSEEK_API_KEY`
and must not downgrade to offline.

Modes:

- `offline`: ruff, pytest, build, manifest checks, no READY claim.
- `auto`: live benchmark when API key exists, otherwise offline gate.
- `release`: live benchmark plus baseline regression gate; missing API key is a
  hard failure.
