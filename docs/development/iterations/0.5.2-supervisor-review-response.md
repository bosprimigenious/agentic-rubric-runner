目标：让 LoopPilot 的一次 run 能交给 agent 评分，而不是只在人读文档里对齐。

Required changes on agent side:

- accept `evaluation_request.json`;
- produce `evaluation_result.json`;
- map score/gate into `pass`, `needs_review`, `blocked`;
- fail closed on missing artifacts or invalid request;
- include evaluator version and manifest IDs.

Suggested command:

```bash

  --request evaluation_request.json \
  --out evaluation_result.json
```

## Shared Gate Rule / 共同门控规则

No side may claim READY unless the relevant aggregate gate produces READY.

任何一边都不能在聚合门控未输出 READY 时宣称 READY。

Allowed wording:

```text
0.5.2.1 implemented; aggregate 0.5.2 NOT READY pending evaluator bridge.
0.5.2.4 web evidence policy documented; web evidence gate NOT READY.
LoopPilot 0.5.2 artifact contract in progress; real agents BLOCKED until 0.6.
```

Forbidden wording:

```text
0.5.2 READY
web evidence implemented
real agents connected
unattended production ready
```

unless acceptance evidence exists.
